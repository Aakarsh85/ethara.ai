Getting Started

  1. 1

    Start the backend server

    Run cd backend && npm run dev in your terminal. Requires MongoDB running locally or an Atlas URI in .env.

  2. 2

    Create an account

    Click Sign Up on the dashboard login page or inside the Chrome extension popup.

  3. 3

    Load the Chrome extension

    Go to chrome://extensions/, enable Developer mode, click Load unpacked, and select the extension/ folder.

  4. 4

    Start saving bookmarks

    Click the extension icon and use the Quick Add button to save the current tab, or manually enter a title and URL.

Frequently Asked Questions

Where is my data stored? expand_more

All bookmarks are stored in your MongoDB database. The JWT token is kept in localStorage (dashboard) and chrome.storage.local (extension). No data is sent to any third party.

Why is the extension not syncing? expand_more

Make sure the backend server is running on port 5000. Check that API_BASE in extension/api.js matches your server address. Reload the extension after any changes.

How do I export my bookmarks? expand_more

Go to Settings → Data Management → Export as JSON. This downloads all your bookmarks as a .json file you can import later or keep as a backup.

How do I deploy to production? expand_more

Deploy the backend to Railway, Render, or Fly.io. Host the dashboard on Vercel or Netlify. Update API_BASE in both the extension and dashboard to your deployed backend URL. Update host_permissions in manifest.json accordingly.

Can I use MongoDB Atlas instead of local MongoDB? expand_more

Yes. Create a free cluster at mongodb.com/atlas, copy the connection string, and paste it as MONGO_URI in backend/.env.
